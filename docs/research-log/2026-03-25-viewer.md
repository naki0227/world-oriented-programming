# Research Log: 2026-03-25 Viewer

## Session Goal

Create the first interactive viewer for `sekai` world snapshots.

## Context

The project already had text output, JSON snapshots, and static figure generation.
The next step was to make world evolution directly inspectable through a minimal interactive interface.

## Actions Taken

- added a browser-based viewer with no external JavaScript dependencies
- implemented JSON loading through file input and sample loading
- added snapshot scrubbing, playback, and projection switching
- added a 3D perspective mode with camera controls
- exposed per-snapshot object state in a sidebar
- generated built-in viewer samples for successful scenarios
- extended the viewer path to handle contradiction reports as structured error states
- added a minimal diagram-aware editing mode that generates `sekai` DSL drafts
- added automatic constraint candidate suggestion for floor reflection and sphere-sphere collision
- extended candidate generation with forbidden-region and speed-limit suggestions
- added the first local round-trip path from generated draft DSL back into the runtime

## Expected Value

- faster inspection of runtime behavior
- a stronger demo story for the project
- a practical stepping stone toward a richer future visual environment
- direct evidence for the claim that diagram construction can become executable without leaving the interface

## Verification Note

- runtime tests remained green with `cargo test`
- sample reports were generated for `bounce`, `two_body_collision`, and the contradiction case `forbidden_region`
- runtime tests remained green with `cargo test`
- the round-trip server script passes Python syntax validation
- local HTTP serving still could not be verified inside the current sandbox because binding a local port was denied

## Next Recommended Step

Verify the round-trip flow in a browser, capture paper screenshots, and then decide whether the next step should be richer 3D rendering, editable constraints, or stronger diagram-to-logic extraction.
