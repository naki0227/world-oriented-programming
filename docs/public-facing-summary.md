# Public-Facing Summary

## One-Sentence Description

`sekai` is an experimental world-oriented programming language where programs describe executable worlds through space, time, and constraints rather than through instruction sequences.

## Short Summary

Most programming systems are organized around procedures, updates, and explicit state mutation.
`sekai` explores a different center of gravity: programs define worlds, and execution is treated as the evolution of an admissible world over time.

The current project already includes:

- a prototype DSL and runtime
- a browser-based viewer
- diagram-to-logic round-trip support
- a first semantics trajectory
- underdetermined-world convergence slices
- comparative evaluation scaffolding
- and a seed paper in LaTeX

## 30-Second Version

`sekai` is not primarily about new syntax.
It is about a new execution model for simulations, geometry-rich worlds, and constraint-driven systems.
Instead of writing frame updates, users define entities, relations, temporal development, and world laws directly.
The runtime then produces admissible world evolution, contradictions, and observations from that model.

## Research Framing

The project currently sits between:

- an idea paper
- a prototype language
- and an emerging operational model

The seed-paper argument is that this representational shift is already coherent enough to support:

- executable examples
- a visible semantics trajectory
- a viewer round-trip
- a first richer-geometry pillar based on visibility
- and a compact comparative evaluation against imperative baselines
